# 「🦞哥眼不聋」微信群AI Bot 产品需求文档

> 这份PRD的核心思想就一句话：Bot是群成员，不是客服。

飞书文档（完整版）：https://feishu.cn/docx/NccrdjtPAolBYFxltBJcm3yknZu

---

## 技术方案（最终版）

- **意图识别**：Claude 3.5 Haiku（便宜快速）
- **正式回复**：统一 Claude Opus（不做分级）
- **被@**：100% 回复，跳过意图识别直接调 Opus
- **部署**：RackNerd VPS（纽约）
- **微信接入**：OpenClaw + @canghe/openclaw-wechat 插件
- **飞书**：文档/知识库载体（不是聊天机器人）

## 架构

```
消息进来
  → 规则前筛（纯表情/图片/系统通知 → 跳过）
  → 被@ ? → 直接调 Opus 回复
  → 关键词命中 ? → Haiku 判断是否需要回复
  → 需要回复 → Opus 生成回复
  → 深度问题 → Opus 生成回复 + 飞书文档
```

## 监控窗口机制

- @虾哥后开启 5 分钟监控窗口
- 窗口内每 30 秒扫描新消息
- 检测追问/补充/反驳 → 继续回复
- 窗口外 → 只在关键词触发时用 Haiku 判断

## 飞书功能

- 深度技术分析 → 飞书文档
- 每周群精华整理 → 飞书文档
- 每月群报告 → 飞书文档
- 成本报告 → 飞书推送给管理员
- 新人指南 → 飞书文档
- 知识库 → 飞书知识空间
